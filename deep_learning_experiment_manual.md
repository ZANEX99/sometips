# 深度学习实验手册


---

## 0.手册目标

深度学习实验不是简单运行代码，也不是随机堆叠模型结构。一个较完整的深度学习实验通常包含以下过程：

```text
理解任务 → 建立Baseline → 提出改进假设 → 设计实验 → 记录结果
→ 对比分析 → 消融验证 → 可视化解释 → 形成技术叙事
```
1. 深度学习实验的基本流程；
2. 常见术语的含义；
3. 如何设计一个可复现的实验；
4. 如何判断模型是否有效；
5. 如何从实验结果中提炼合理的研究故事；
6. 如何规范整理代码、结果和实验报告。

需要强调的是，深度学习模型具有一定黑盒特征，因此实验验证非常重要。但实验验证不等于盲目试错。较好的实验应当是**假设驱动的系统探索**：先提出一个可解释的结构假设，再通过对照实验和消融实验判断该假设是否成立。

## 1.深度学习实验的基本认识

### 1.1为什么深度学习需要大量实验？

深度学习模型通常具有大量参数，模型性能会受到多种因素影响，例如：

- 数据集规模和质量；
- 模型结构；
- 损失函数；
- 优化器；
- 学习率；
- Batch size；
- 训练轮数；
- 数据增强方式；
- 正则化方法；
- 评价指标；
- 随机种子。

因此，深度学习研究通常需要通过多组实验寻找较合适的模型结构和训练策略。

### 1.2实验不是随机试错

不推荐的实验方式：

```text
随便换模型 → 看哪个结果高 → 直接写结论
```

推荐的实验方式：

```text
发现问题 → 提出假设 → 修改结构 → 设计对照 → 分析结果 → 修正假设
```

例如：

```text
问题：普通CNN在细小纹理分类任务上表现不好。
假设：模型对局部边缘和纹理细节的表达能力不足。
改进：加入边缘增强模块或注意力模块。
验证：与原始CNN进行对比，并设计消融实验。
分析：观察准确率、训练曲线、误分类样本和特征图变化。
```

## 2.常见术语解释

### 2.1数据相关术语

| 术语 | 含义 | 简单解释 |
|---|---|---|
| Dataset | 数据集 | 用于训练和测试模型的数据集合 |
| Sample | 样本 | 数据集中的一个数据点，例如一张图片 |
| Label | 标签 | 样本对应的真实类别或目标值 |
| Training set | 训练集 | 用于训练模型的数据 |
| Validation set | 验证集 | 用于调参和选择模型的数据 |
| Test set | 测试集 | 用于最终评价模型泛化能力的数据 |
| Data augmentation | 数据增强 | 通过翻转、裁剪、旋转等方式扩充训练数据 |

### 2.2模型相关术语

| 术语 | 含义 | 简单解释 |
|---|---|---|
| Model | 模型 | 从输入到输出的映射函数 |
| Parameter | 参数 | 模型内部通过训练自动学习的变量 |
| Hyperparameter | 超参数 | 人为设置的参数，例如学习率和Batch size |
| Backbone | 主干网络 | 模型中的主要特征提取部分 |
| Module | 模块 | 插入模型中的功能结构，例如注意力模块 |
| Classifier | 分类器 | 将特征映射到类别结果的部分 |
| Feature | 特征 | 模型从数据中提取的有用表示 |

### 2.3训练相关术语

| 术语 | 含义 | 简单解释 |
|---|---|---|
| Epoch | 训练轮数 | 模型完整看完一遍训练集 |
| Batch | 批次 | 每次送入模型的一组样本 |
| Batch size | 批大小 | 每个Batch包含的样本数量 |
| Iteration | 迭代次数 | 模型完成一次参数更新 |
| Loss | 损失 | 衡量预测结果和真实标签之间的差距 |
| Optimizer | 优化器 | 用于更新模型参数的方法 |
| Learning rate | 学习率 | 每次参数更新的步长 |
| Backpropagation | 反向传播 | 根据损失计算梯度并更新参数的过程 |

### 2.4评价相关术语

| 术语 | 含义 | 常见任务 |
|---|---|---|
| Accuracy | 准确率 | 分类任务 |
| Precision | 精确率 | 分类、检测任务 |
| Recall | 召回率 | 分类、检测任务 |
| F1-score | F1值 | 类别不平衡任务 |
| mAP | 平均精度均值 | 目标检测任务 |
| mIoU | 平均交并比 | 图像分割任务 |
| PSNR | 峰值信噪比 | 图像恢复任务 |
| SSIM | 结构相似性 | 图像恢复任务 |
| MAE | 平均绝对误差 | 回归和预测任务 |
| RMSE | 均方根误差 | 回归和预测任务 |

## 3.深度学习实验基本流程

一个完整的深度学习实验可以拆分为以下步骤：

```text
任务定义 → 数据准备 → 模型构建 → 损失函数设置 → 优化器设置
→ 模型训练 → 模型验证 → 模型测试 → 结果记录 → 结果分析
```

### 3.1任务定义

首先要明确实验任务属于哪一类。

| 任务类型 | 输入 | 输出 | 常用指标 |
|---|---|---|---|
| 图像分类 | 图像 | 类别标签 | Accuracy、F1-score |
| 目标检测 | 图像 | 目标框和类别 | mAP |
| 语义分割 | 图像 | 像素级类别 | mIoU、Dice |
| 图像恢复 | 退化图像 | 清晰图像 | PSNR、SSIM |
| 文本分类 | 文本 | 类别标签 | Accuracy、F1-score |
| 时间序列预测 | 历史序列 | 未来数值 | MAE、RMSE |

### 3.2数据准备

数据准备通常包括：

1. 下载或整理数据集；
2. 划分训练集、验证集和测试集；
3. 检查标签是否正确；
4. 统一数据尺寸；
5. 进行归一化处理；
6. 根据任务需要进行数据增强。

需要注意：

- 测试集不能参与训练；
- 验证集用于调参，测试集用于最终评价；
- 不要让同一对象同时出现在训练集和测试集中；
- 数据划分方式应当在实验报告中说明。

### 3.3模型构建

模型构建需要说明：

1. 输入是什么；
2. 模型由哪些部分组成；
3. 每个模块的作用是什么；
4. 输出是什么；
5. 模型为什么适合当前任务。

简单分类模型可以按如下方式描述：

```text
该模型由特征提取部分和分类部分组成。
特征提取部分通过卷积层提取图像中的局部纹理、边缘和形状信息。
分类部分通过全连接层将特征映射为类别概率。
最终模型输出每个类别的预测概率。
```

复杂模型可以按如下问题进行解释：

```text
该模块输入什么？
该模块对特征做了什么变换？
该模块输出什么？
该模块为什么可能有效？
该模块会增加什么计算代价？
```

### 3.4损失函数设置

损失函数用于衡量模型预测结果和真实结果之间的差距。

| 任务 | 常用损失函数 |
|---|---|
| 单标签分类 | CrossEntropyLoss |
| 多标签分类 | BCEWithLogitsLoss |
| 回归任务 | MSELoss、L1Loss |
| 图像恢复 | L1Loss、MSELoss、感知损失 |
| 语义分割 | CrossEntropyLoss、DiceLoss |

### 3.5优化器设置

常见优化器包括：

| 优化器 | 特点 |
|---|---|
| SGD | 稳定、经典，但调参要求较高 |
| SGD+Momentum | 在SGD基础上加入动量，加速收敛 |
| Adam | 自适应学习率，初学实验常用 |
| AdamW | 对权重衰减处理更合理，Transformer类模型常用 |

初学者建议从Adam或AdamW开始。

### 3.6训练与测试

训练阶段：

```text
读取训练数据 → 前向传播 → 计算Loss → 反向传播 → 更新参数
```

测试阶段：

```text
读取测试数据 → 前向传播 → 计算评价指标 → 保存结果
```

测试时需要注意：

- 使用`model.eval()`；
- 不进行梯度更新；
- 通常使用`torch.no_grad()`；
- 不要在测试集上调参。

## 4.Baseline意识

### 4.1什么是Baseline？

Baseline是基础对照模型。它用于回答一个关键问题：

```text
新方法到底有没有比原始方法更好？
```

如果没有Baseline，就无法判断改进是否有效。

### 4.2Baseline实验示例

以图像分类为例：

| 实验编号 | 模型 | 实验目的 |
|---|---|---|
| Exp-001 | 普通CNN | 建立基础性能 |
| Exp-002 | CNN+Dropout | 验证Dropout是否缓解过拟合 |
| Exp-003 | CNN+BatchNorm | 验证归一化是否改善训练稳定性 |
| Exp-004 | CNN+Attention | 验证注意力机制是否增强特征表达 |
| Exp-005 | CNN+Residual | 验证残差连接是否改善深层训练 |

应先完成Baseline，再逐步增加改进模块。

## 5.消融实验

### 5.1什么是消融实验？

消融实验是指去掉模型中的某个模块，观察性能变化。

其目的不是单纯比较分数，而是回答：

```text
这个模块是否真的有贡献？
```

### 5.2消融实验示例

| 方法 | Accuracy |
|---|---|
| Baseline CNN | 91.2% |
| CNN+Module A | 93.1% |
| CNN+Module B | 92.4% |
| CNN+Module A+Module B | 94.0% |

可以得到如下分析：

```text
与Baseline相比，Module A带来1.9%的性能提升，Module B带来1.2%的性能提升。
当两个模块同时使用时，模型取得最高Accuracy，说明二者可能具有一定互补作用。
如果去掉Module A后性能明显下降，则说明Module A对最终结果具有重要贡献。
```

### 5.3消融实验注意事项

1. 每次只改变一个主要因素；
2. 保持数据集、训练轮数、优化器和学习率尽量一致；
3. 不要同时改变多个设置，否则无法判断提升来自哪里；
4. 记录每次实验的完整参数；
5. 重要实验建议重复多次并报告平均结果。

## 6.调参方法

### 6.1调参不是玄学

调参应当按照优先级进行，而不是随意尝试。

推荐顺序：

```text
学习率 → Batch size → 模型容量 → 正则化方法 → 数据增强 → 训练轮数
```

### 6.2学习率

学习率通常是最重要的超参数之一。

| 现象 | 可能原因 | 解决方法 |
|---|---|---|
| Loss剧烈震荡 | 学习率过大 | 降低学习率 |
| Loss下降很慢 | 学习率过小 | 适当增大学习率 |
| 训练出现NaN | 学习率过大或数据异常 | 降低学习率并检查数据 |
| 前期下降快后期停滞 | 学习率后期偏大 | 使用学习率衰减策略 |

### 6.3Batch size

| 现象 | 可能原因 | 解决方法 |
|---|---|---|
| 显存不足 | Batch size过大 | 减小Batch size |
| 训练波动较大 | Batch size过小 | 适当增大Batch size |
| 泛化效果变差 | Batch size过大 | 适当减小Batch size或调整学习率 |

### 6.4模型容量

| 现象 | 可能原因 | 解决方法 |
|---|---|---|
| 训练集准确率低 | 模型表达能力不足 | 增加层数或通道数 |
| 训练集好、测试集差 | 过拟合 | 加入正则化或数据增强 |
| 训练速度过慢 | 模型过大 | 减少参数或使用轻量结构 |

## 7.训练曲线分析

训练曲线是判断模型状态的重要依据。

### 7.1正常训练

一般表现为：

- 训练Loss下降；
- 验证Loss下降；
- 训练Accuracy上升；
- 验证Accuracy上升；
- 两者差距不大。

### 7.2过拟合

典型表现：

- 训练Loss继续下降；
- 验证Loss开始上升；
- 训练Accuracy很高；
- 测试Accuracy明显低于训练Accuracy。

可能解决方法：

1. 增加数据增强；
2. 使用Dropout；
3. 使用Weight decay；
4. 减小模型规模；
5. 使用Early stopping；
6. 增加训练数据。

### 7.3欠拟合

典型表现：

- 训练Loss和验证Loss都较高；
- 训练Accuracy和验证Accuracy都较低；
- 模型没有充分学习训练数据。

可能解决方法：

1. 增加模型容量；
2. 增加训练轮数；
3. 调整学习率；
4. 更换更合适的模型结构；
5. 检查数据和标签是否存在问题。

## 8.实验记录规范

### 8.1为什么要记录实验？

深度学习实验容易受到多种因素影响。如果不记录实验过程，后期很难判断：

- 哪个模型有效；
- 哪个参数有效；
- 哪次实验结果可复现；
- 失败原因是什么；
- 下一步应该如何改进。

### 8.2实验记录表模板

| 项目 | 内容 |
|---|---|
| 实验编号 | Exp-001 |
| 实验日期 | 2026-XX-XX |
| 实验任务 | 图像分类 |
| 数据集 | MNIST |
| 模型 | SimpleCNN |
| 改进内容 | 加入Dropout |
| Learning rate | 0.001 |
| Batch size | 64 |
| Epoch | 20 |
| Optimizer | Adam |
| Loss | CrossEntropyLoss |
| 训练Accuracy | 99.1% |
| 测试Accuracy | 98.4% |
| 训练现象 | Loss下降稳定 |
| 问题分析 | 存在轻微过拟合 |
| 下一步计划 | 尝试数据增强或Weight decay |

### 8.3实验结果命名建议

建议按照如下格式保存实验结果：

```text
results/

│   ├── config.yaml

│   ├── acc_curve.png

│
├── Exp002_cnn_dropout/
│   ├── config.yaml
│   ├── train_log.txt
│   ├── loss_curve.png
│   ├── acc_curve.png
```

## 10.可复现性要求

为了保证实验结果可信，建议每个实验至少满足以下要求：

### 10.1固定随机种子

```python
import random
import numpy as np
import torch

def set_seed(seed=42):
    random.seed(seed)
    np.random.seed(seed)
    torch.manual_seed(seed)
    torch.cuda.manual_seed_all(seed)
```

### 10.2记录环境

需要记录：

```text
Python版本
PyTorch版本
CUDA版本
GPU型号
操作系统
主要依赖库版本
```

### 10.3保存配置

每次实验都应保存配置文件，例如：

```yaml
experiment_name: Exp001_baseline
dataset: MNIST
model: SimpleCNN
epochs: 20
batch_size: 64
learning_rate: 0.001
optimizer: Adam
loss: CrossEntropyLoss
seed: 42
```

### 10.4保存模型权重

建议保存：

```text
best_model.pth
last_model.pth
```

其中：

- `best_model.pth`表示验证集性能最好的模型；
- `last_model.pth`表示最后一个Epoch训练得到的模型。

### 10.5重复实验

如果实验结果波动较大，应重复多次并报告平均值和标准差：

```text
Accuracy: 92.35% ± 0.21%
```

## 11.结果可视化

### 11.1必须绘制的基础曲线

建议每个实验至少保存：

1. Training loss曲线；
2. Validation loss曲线；
3. Training accuracy曲线；
4. Validation accuracy曲线。

### 11.2分类任务常见可视化

| 可视化方式 | 作用 |
|---|---|
| 混淆矩阵 | 查看哪些类别容易混淆 |
| 错误样本展示 | 分析模型失败原因 |
| 特征图可视化 | 观察卷积层学习到的响应 |
| t-SNE或PCA可视化 | 观察特征分布是否可分 |
| Grad-CAM | 查看模型关注区域 |

### 11.3图像恢复任务常见可视化

| 可视化方式 | 作用 |
|---|---|
| 输入图像 | 展示退化图像 |
| 输出图像 | 展示恢复结果 |
| 真实图像 | 作为参考结果 |
| 残差图 | 查看错误区域 |
| 局部放大图 | 比较细节恢复能力 |

## 12.从实验到技术叙事

### 12.1什么是技术叙事？

技术叙事不是编故事，而是把以下内容组织成一个合理逻辑：

```text
任务问题 → 方法动机 → 结构设计 → 实验验证 → 结果解释 → 局限分析
```

### 12.2常见叙事结构

可以按照如下方式组织实验报告或论文段落：

```text
首先，Baseline在当前任务上取得了一个基础结果。
但是，从训练曲线和错误样本可以观察到，模型在某些类别或某些细节区域上仍然存在不足。
针对这一问题，我们引入了某个改进模块，希望增强模型对相关特征的表达能力。
实验结果显示，改进模型在测试集上取得了更好的性能。
进一步的消融实验表明，当去除该模块后，性能出现下降，说明该模块对最终结果具有贡献。
同时，可视化结果也显示，改进模型能够更关注关键区域。
```

### 12.3结果分析不能只写“效果更好”

不推荐：

```text
改进模型效果更好，说明该方法有效。
```

推荐：

```text
与Baseline相比，改进模型的Accuracy由91.2%提升至93.1%。
从训练曲线可以看出，改进模型在前10个Epoch内收敛速度更快，并且验证集Accuracy更加稳定。
消融实验进一步表明，去除该模块后Accuracy下降至91.8%，说明该模块对特征表达具有实际贡献。
```

## 13.常见问题排查

| 问题 | 可能原因 | 解决方法 |
|---|---|---|
| 显存不足 | Batch size过大，模型过大 | 减小Batch size或模型规模 |
| Loss不下降 | 学习率不合适，标签错误 | 检查学习率和数据标签 |
| Accuracy一直很低 | 输出类别数错误，标签编码错误 | 检查最后一层输出维度 |
| 训练速度很慢 | 数据加载慢，模型复杂 | 增加num_workers或简化模型 |
| 测试结果异常高 | 训练集和测试集混淆 | 检查数据划分 |
| 每次结果差异大 | 未固定随机种子 | 设置随机种子并记录环境 |
| 出现NaN | 学习率过大，梯度爆炸 | 降低学习率或使用梯度裁剪 |
| 训练集好测试集差 | 过拟合 | 数据增强、Dropout、Weight decay |
| 训练集和测试集都差 | 欠拟合 | 增加模型容量或训练轮数 |

## 18.初学者建议

1. 先跑通Baseline，再考虑改进；
2. 每次只改一个主要因素；
3. 不要只看最终Accuracy，也要看训练曲线；
4. 不要只保存最好结果，也要记录失败实验；
5. 实验报告中要说明为什么这样设计；
6. 结果不好并不等于实验失败，关键是要分析原因；
7. 代码能运行只是第一步，能解释结果才是科研训练的重点。

## 19.一句话总结

深度学习实验的核心不是“把模型跑起来”，而是通过规范实验验证一个技术假设，并把任务问题、模型设计、实验结果和结果解释组织成可信的研究逻辑。
